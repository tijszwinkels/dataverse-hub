# Handoff: Hub Server Sub-Package Refactoring

## Goal

Refactor the dataverse hub server from a flat `package main` (16 source files, ~3,725 LOC) into sub-packages with clear boundaries: `object/`, `realm/`, `storage/`, `auth/`, `upstream/`, `serving/`. This separates concerns, deduplicates code between Hub (root mode) and Proxy, and prepares the `realm/` package for future per-realm configuration (per-realm access control, per-realm upstream routing).

## Key Decisions

These were discussed and agreed upon but worth highlighting:

1. **`InField` + `IsPubkeyRealm` + `PubkeyRealms` stay in `object/`**, not `realm/`. Reason: `VerifyEnvelope` (in `object/`) calls `PubkeyRealms`, so putting them in `realm/` would create an import cycle. `IsPubkeyRealm` is a format check (base64url decode + byte check), not access control logic.

2. **`realm/` is intentionally small** (just `IsPublicObject` + `HasMatchingRealm` today). The user envisions it growing into a per-realm config registry (who has access, which upstream per realm). The package boundary is set up now for that future.

3. **`auth/` gets its own `writeAuthError`** (3-line helper using `object.APIError`) rather than importing `serving/` or creating an extra `httputil/` package. Avoids coupling.

4. **`SyncPending` needs `Upstream.DirectDo(req)`** — a new method that calls the HTTP client directly without the availability fast-fail check. Currently `syncpending.go` accesses the unexported `upstream.client` field.

5. **`isUpstreamDown`** moves from `proxy.go` to `upstream/client.go` as `IsUpstreamDown`. It's used by both `upstream.go`'s HealthCheck and `proxy.go`.

6. **Deduplication targets**: `paginateAndLoad`, `enrichWithInboundCounts`, router setup, and content negotiation logic are all duplicated between Hub and Proxy. These become shared functions in `serving/`.

7. **Integration tests stay at root** (`package main`). Unit tests move with their packages. Shared test helpers (`testKeypair`, `signChallenge`, `doPut`, `doGet`, `signedObject`, `loadTestFixture`) go into a new `testutil_test.go` at root.

8. **Atomic moves required**: Go doesn't allow duplicate symbols, so each file move must atomically create the sub-package file + update all callers + delete the original. Can't have both old and new coexisting.

## Implementation Plan

8 phases, bottom-up. Commit after each phase for rollback safety.

### Phase 1: `object/` — Types + Crypto
- Create `object/types.go`: all types from `types.go` except `Config`, plus `IsPubkeyRealm`/`PubkeyRealms` from `realm.go`
- Create `object/verify.go`: everything from `verify.go`, export `canonicalJSON` → `CanonicalJSON`, `decompressP256` → `DecompressP256`
- Move `verify_test.go` → `object/verify_test.go`, create `object/types_test.go`
- Symlink `object/testdata → ../testdata`
- Update ALL root files to use `object.Xxx` references
- Delete `types.go`, `verify.go` from root

### Phase 2: `realm/` — Access Control
- Create `realm/realm.go`: `IsPublicObject`, `HasMatchingRealm` (imports `object`)
- Move relevant tests from `realm_test.go`
- Delete `realm.go` from root

### Phase 3: `storage/` — Store + Index
- Create `storage/store.go` from `store.go` (no sub-package deps)
- Create `storage/index.go` from `index.go` (imports `object/`, `realm/`), includes `InboundFilters`
- Move `store_test.go` → `storage/store_test.go`
- Delete originals

### Phase 4: `auth/` — Authentication + Rate Limiting + Widget
- Create `auth/auth.go` from `auth.go` — replace `writeError` with local `writeAuthError`
- Create `auth/ratelimit.go` from `ratelimit.go`
- Create `auth/widget.go` + `auth/widget_embed.go` from `authwidget.go` + `widget_embed.go`
- Copy `widget.html` → `auth/widget.html`
- Move test files
- Delete originals

### Phase 5: `upstream/` — HTTP Client + SyncPending
- Create `upstream/client.go` from `upstream.go` + `IsUpstreamDown` from `proxy.go`
- Add `DirectDo` method
- Create `upstream/sync.go` from `syncpending.go`
- Move test files
- Delete originals

### Phase 6: `serving/` — HTTP Handlers (biggest phase)
- Create `serving/helpers.go`: `WriteError`, `WriteList`, `PaginateAndLoad`, `EnrichWithInboundCounts`, `ParseLimit`, `ParseCursor`, `JsonContentType`
- Create `serving/content.go`: `AcceptsHTML`, `AcceptsMimeType`, `ServeBlob`, `StripBlobData`, `ExtractHTML`, `PageETagSuffix`, `ExtractSortKey`
- Create `serving/router.go`: `BuildRouter` (deduplicated)
- Create `serving/hub.go`: Hub struct + root-mode handlers
- Create `serving/proxy.go`: Proxy struct + proxy-mode handlers
- Delete `handlers.go`, `server.go`, `proxy.go` from root

### Phase 7: Update Root
- Update `main.go` to import and wire sub-packages
- Create `testutil_test.go` with shared test helpers
- Update all integration test files
- `go test ./...`

### Phase 8: Final Cleanup
- `go vet ./...`, `go test -race ./...`
- Verify Docker build
